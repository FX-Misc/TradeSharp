Редактирование сделок

Интерфейс Trade# server administration предоставляет возможность администратору редактировать как открытые, так и закрытые сделки.

Для того, чтобы осуществить редактирование, на странице «Позиции», в первом столбце таблицы нужно выделить сделки, которые необходимо редактировать и затем пройти по ссылке «редактировать выбранные сделки».

Редактирование выбранных сделок осуществляется на странице «Редактирование позиций». На этой странице может быть отображено до трёх таблиц:

  1. Открытые позиции в выборке – отображается, если среди позиций, попавших в список на редактирование, есть открытые позиции. Таблица содержит свойства открытых позиций, которые разрешено редактировать.
  2. Закрытые позиции в выборке – отображается, если среди позиций, попавших в список на редактирование, есть закрытые позиции. Таблица содержит свойства закрытых позиций, которые разрешено редактировать.
  3. Остальные позиции в выборке – отображается, если среди позиций, попавших в список на редактирование, есть позиции со статусом «Открывается», «Закрывается».

Каждой таблице присущ свой набор свойств, которые можно редактировать. На прошлом рисунке, в качестве примера, были отмечены для редактирования одна открытая позиция и две закрытые. На следующем рисунке приведён пример редактирования этих трёх позиций.

Обратите внимание, что в данном примере, некоторые свойства у всех закрытых позиций имеют одинаковые значения (например, свойство StopLoss). В этом случае в поле значения сразу отображается значения StopLoss (т.к. во всех элементах они одинаковые).

В последнем столбце отображается комментарий о тех значениях, которые сейчас содержаться в соответствующем свойстве редактируемых элементах. В приведённом примере, значения свойства «Время входа», в двух редактируемых закрытых позициях, разные. Поэтому, в комментарии к этому поля перечислены все эти значения, через запятую.

Важно отметить, что при редактировании какого либо свойства результат сохранится для всех соответствующих позиций. Например, при редактировании свойства «Время входа» в таблице «Закрытые позиции в выборке», новое значение этого свойства будет установлено для всех элементов в выборке (в этом примере – для двух позиций). Разумеется, одноимённое свойство у открытых позиций изменено не будет. Изменения коснуться только закрытых позиций, потому, что они вносились в таблице «Закрытые позиции в выборке».
