Часто задаваемые вопросы

Оглавление

1 Установка и первый запуск терминала

Процесс "нормального" запуска терминала подробно описан в соответствующем разделе справочной системы.

1.1 При скачивании дистрибутива терминала браузер сообщает:

Не рекомендуется загружать файл TradeSharp_client.zip, так как он может представлять опасность.
Вы можете увидеть это или подобное сообщение при определенных настройках безопасности вашего интернет-браузера. Причина заключается в отсутствии в составе ПО терминала цифровой подписи авторизованного удостоверяющего центра. Скачивать терминал или нет - решать Вам. Мы можем только уверить Вас в том, что, разрабатывая торговый терминал Trade#, не ставили перед собой задачи внедрения вредоносного кода, способного навредить Вашему ПК или похитить с него конфиденциальную информацию.

Окно браузера Google Chrome

Решение проблемы

Выберите опцию сохранения файла. Для браузера Google Chrome (см рисунок) нажмите кнопку справа от кнопки "Отменить передачу" и выберите команду Сохранить.

1.2 Я скачал файл TradeSharp_client.zip, распаковал, программа не запускается

Торговый терминал Trade# запускается вызовом менеджера обновлений - файл TradeSharp.UpdateManager.exe. Кроме этого, в каталоге терминала обязательно должны быть следующие файлы:
TradeSharp.UpdateManager.exe.config, log4net.dll, TradeSharp.UpdateContract.dll.

Все файлы на месте, кликаю по названию файла два раза, ничего не происходит.
Возможно, у Вас не установлена версия Microsoft .NET Framework 4.0 или новее. MS .NET Framework - бесплатно распространяемая Microsoft платформа, необходимая для работы многих программ. Скачать и установить платформу можно по этой ссылке.

Файл запускается, но завершается с ошибкой. Причин тому может быть множество. С наибольшей вероятностью, ПО Trade# не располагает достаточными правами для модификации содержимого своей директории. Например, для того, чтобы обновить файлы терминала. Вы можете произвести соответствующие настройки безопасности вашей операционной системы. Вы также можете запустить терминал от имени администратора. Для этого нажмите правую кнопку мыши на файле TradeSharp.UpdateManager.exe. В открывшемся меню выберите пункт Запуск от имени администратора.

Отсутствует интернет-соединение, терминал не запускается.
Терминал может функционировать при отсутствии интернет-соединения. Убедитесь в наличии файлов TradeSharp.client.exe, TradeSharp.Client.exe.config. Убедитесь в том, что размер каждого из файлов в каталоге терминала не равен 0 байт - следует обращать внимание на файлы с расширением dll, exe.

1.3 При повторных запусках терминал сообщает об ошибках в работе

Убедитесь в том, что вы не пытаетесь запустить вторую и последующие копии программы из одной и той же папки (кликая два раза на один и тот же файл / ярлык). На одном ПК могут быть запущены несколько копий терминала Trade#, однако, в таком случае, они должны находиться в разных каталогах. Терминал может быть запущен и свернут в системный трей (см панель задач Windows, в нижней части экрана). Чтобы убедиться, что нет других запущенных копий терминала, вы можете вызвать диспетчер задач. (Ctrl + Alt + Delete). В диспетчере задач выберите вкладку Процессы, нажмите кнопку "Отображать процессы всех пользователей". Ищите процесс TradeSharp.Client.exe.

1.4 Не могу закрыть терминал

Возможно, в терминале Trade# открыто окно диалога - обратите внимание на панель задач Windows (внизу экрана).

1.5 Терминал не обновляется

Обратите внимание - запускать терминал следует исполняемым файлом TradeSharp.UpdateManager.exe. При запуске появляется окно обновления:

Окно обновления

Окно обновления не появляется. Возможно, окно обновлений появилось и незамедлительно закрылось. Проверьте актуальность версии терминала - меню Справка | О программе. Также, возможно, произошла ошибка при попытке получить обновления. Вы можете сообщить в службу поддержки. Желательно при этом к сообщению об ошибке приложить файл upmanager_log.txt.

Окно обновления появилось, отобразился список обновляемых файлов, но загрузки не происходит. Список файлов меняет цвет на красный. Возможная причина - у программы обновления нет доступа к изменению файлов в своем каталоге (обновлению старых файлов). Такое может произойти, если, например, терминал установлен в каталоге, защищенном операционной системой (например, Program Files). Либо настройки безопасности операционной системы препятствуют перезаписи файлов. Также, возможно, из этого же каталога уже запущена другая копия терминала. См пп 1.2, 1.3.

1.6 Файл справки пуст (чистые страницы)

Убедитесь, что файл справки terminal.chm имеет ненулевую длину (порядка 1 Мб и более). Если файл пуст, см. раздел 1.2.

Нажмите правую кнопку мыши на имени файла terminal.chm. В открывшемся контекстном меню выберите нижний пункт - Свойства. В окне свойств файла, внизу окна, вы можете увидеть предупреждение вида:

Осторожно: Этот файл получен с другого компьютера и, возможно, был заблокирован с целью защиты компьютера.

Нажмите кнопку Разблокировать - разделы справочной системы станут доступными.

1.7 Прочие ошибки

Обратите внимание на файлы log-file.txt, upmanager_log.txt. В этих файлах записи об ошибок чередуются с отладочными сообщениями. Каждая запись содержит преффикс INFO, ERROR, .... Записи об ошибках начинаются с ERROR. В сообщении об ошибке может содержаться подсказка к ее устранению. Сообщая нам об ошибке, вы можете вложить в письмо / сообщение мессенджера эти два файла.

2 Ошибки подключения

2.1 Терминал запущен, но новые котировки не приходят

Обратите внимание на надпись в правом нижнем углу терминала - состояние связи:

Статус подключения

На рисунке текст (Подключен, Не подключен, Ошибка подключения) синий. Если котировки не поступают в терминал, цвет текста меняется на красный. Если цвет текста синий, но котировки не поступают в терминал, проверьте это, открыв окно актуальных котировок - меню Вид | Котировки.

Решение проблемы

Возможно, подключение к источнику котировок (брокеру) блокируется антивирусным ПО или соответствующими службами операционной системы. Вы можете уточнить, какой IP-адрес / порт использует терминал для связи с провайдером. Для этого, откройте файл TradeSharp.Client.exe.config программой Блокнот (Notepad). В файле найдите строки вида

<add key="Quote.ServerPort" value="55056"/>
<add key="Quote.ServerHost" value="70.38.11.49"/>

Убедитесь, что соответствующие адрес / порт не блокируются антивирусным ПО (брандмауэром).

2.2 В статусной строке выводится сообщение - ошибка подключения

При запуске терминала либо в процессе работы может появиться сообщение - "Ошибка сервера". В статусной строке написано:

Ошибка подключения

Решение проблемы

Во-первых, убедитесь в том, что ваша версия терминала Trade# актуальна. Обратите внимание - запускать терминал следует исполняемым файлом TradeSharp.UpdateManager.exe. Если при обновлении возникают проблемы, смотрите раздел 1 п. 5 (обновление терминала).

Возможно, причина ошибки кроется в том, что канал связи терминала с торговым сервером блокируется антивирусным ПО вашего компьютера.

2.3 На графике котировок появился гэп

Возможно, в течение некоторого времени терминал не мог установить связь с сервером и не получал котировки. При следующей загрузке терминала гэп будет обнаружен и заполнен. Вы можете нажать на графике правую кнопку мыши, вызвав контекстное меню:

Гэп на графике котировок

В меню вызовите команду Синхр. котировки.... В открывшемся окне показан интервал сохраненной истории котировок по торгуемому инструменту. Левой кнопкой мыши вы можете перетащить границу интервала, расширив наблюдаемый на графиках диапазон данных. Нажмите кнопку "Принять" - гэп будет заполнен и все графики торгуемого инструмента будут обновлены.
