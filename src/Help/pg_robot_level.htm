Робот "Торговля от уровней"

Робот осуществляет вход в рынок и сопровождение сделок. При проверках, предшествующих входу в рынок, равно как и при сопровождении сделок робот может отталкиваться только от собственных сделок (проверяются по magic), либо от всех сделок по данному счету (задается параметром робота «Учитывать все сделки»).

Вход в рынок

Торговый робот отслеживает ряд условий по нескольким символам (валютным парам). При соблюдении условия робот открывает сделку. Условия задаются в виде:

Продать EURUSD с 19.12.2011 10:15 по 19.12.2011 23:59 если цена выше 1.3730
Купить USDCHF с 19.12.2011 10:15 если цена ниже 0.9255
…

Покупка или продажа производится только на закрытии очередной свечи. Второе условие (время) может быть не указано. Тогда данное условие проверяется только один раз, по первому со времени старта закрытию свечи.
Можно задать одновременно несколько условий, в том числе указать несколько уровней для одного символа (валютной пары).
В настройках робота задается максимальное количество сделок по одной валютной паре.
SIC! Если в настройках робота описано, например, 3 валютные пары, робот будет делить общую доступную для входа в рынок сумму капитала на 3 × макс. сделок по паре частей.

Объем входа и проверка плеча

Отправными точками в расчете объема являются параметры: количество сделок и максимальное плечо. Пример расчета:

Текущий баланс: 1,000,000 USD.
Макс. плечо: 1
Макс. кол.- во сделок: 5, валютных пар задано в настройках: 4.
Валютная пара: EURUSD
Мин. объем: 10,000
Мин. шаг объема: 5,000
Объем = 1 x 1,000,000 / (4 x 5) = 50,000 USD = 50,000 / 1.3750 = 36,360 EUR
С учетом минимального шага объем сделки равен 35,000 EUR.

Если открытие сделки приведет к увеличению суммарного плеча по счету сверх установленного значения, сделка открыта не будет.
Если данный робот (проверяется по параметру сделки - magic) открыл в одном направлении по одному инструменту максимально разрешенное в настройках количество позиций, новая позиция открыта не будет.

Сопровождение сделки

Робот может «защищать» позиции – переносить стоп-ордер позиций счета в точку входа + N пунктов от входа в направлении сделки. «Защита» происходит при прохождении рынком M пунктов в направлении входа. «Защита» может производиться как для каждой отдельно взятой сделки, так и для суммарной по каждому инструменту сделке, иначе говоря – по «медианной» цене.

На рисунке отмечены две продажи – A, B. Если предположить, что объемы сделок A, B одинаковы, «медианная» цена будет равна средней для этих двух сделок цене входа. В общем случае, «медианная» цена высчитывается с учетом направления и объема входа каждой сделки робота.
К «медианной» цене добавляется смещение, заданное параметром «уровень защиты». Если рынок преодолевает этот уровень, каждой сделке выставляется стоп-ордер на отметке «медианной» цены + N пунктов.
